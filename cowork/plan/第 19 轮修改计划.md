# 第 19 轮首页与站点框架整理实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [x]`) syntax for tracking.

**Goal:** 系统整理中文 fork 的主页和站点框架入口，让首页、导航和 contents 弹窗看起来是本地中文课程站，而不是原仓库英文站的直出镜像。

**Architecture:** 优先沿用现有静态 HTML/CSS/vanilla JS 架构，不引入框架。把“原仓库推广”和“本地课程浏览”拆开：站内导航、contents 弹窗和课程入口全部走本地页面；原仓库只保留一个明确按钮。中文文案先覆盖主页和共享框架，再用构建脚本为 contents 弹窗补充中文 phase/lesson 数据。

**Tech Stack:** 静态 HTML、CSS、vanilla JavaScript、`node:test`、Browser/IAB QA、现有 `site/build.js` 数据构建。

---

## Scope

本轮必须覆盖：
- 首页 `site/index.html` 的中文化和原仓库信息清理。
- 首页 contents/phase 弹窗的中文化和本地课程跳转。
- 顶部导航的中文化、GitHub star/count 移除、主题按钮改成日/月图标。
- 中文课程页 `lesson.html` 的侧边栏 phase 导航和课程标题中文化，优先复用 `site/build.js` 产出的 `nameZh` / `descZh`。
- 课程页网页内直接运行代码能力移除：不加载 Pyodide runner，不显示“浏览器可运行”入口，不再把 browser runner 当作课程页功能展示。
- 与首页共享的 header/footer 同类痕迹，同步清理到 `lesson.html`、`catalog.html`、`glossary.html`、`prereqs.html`、`about.html`。

本轮可以延伸但要谨慎：
- 删除 `header.js`，前提是所有 `.header-github` 和 `data-gh-stars` 调用都已经移除。
- 删除 `runner-pyodide.js` / `runner-worker.js`，前提是 `lesson.html`、`cmdpalette.js`、测试和文案均不再引用 browser runner；否则先保留为孤立历史文件但不加载、不展示。

本轮不做：
- 大规模重新设计视觉风格。
- 翻译 503 篇课程正文。
- 重写站点为框架应用。

function readLocalizedLessonTitle(lessonRel, locale) {
  const docPath = path.join(REPO_ROOT, lessonRel, 'docs', `${locale}.md`);

}
