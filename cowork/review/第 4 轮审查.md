# 第 4 轮审查

日期：2026-06-09

范围：
- 复查第 3 轮后的翻译残留。
- 继续优先处理“解释性英文大量残留”的课程。
- 对本地网页保持抽样复验要求：页面能加载，中文内容、公式/图示、代码高亮和测验区域正常。

## 结论

第 4 轮仍有问题，不能结束循环。

第 3 轮修复后的自动校验和本地页面复验已通过，但残留扫描显示 `phases/11-llm-engineering/12-guardrails` 的中文文档和 quiz 仍有大量可自然翻译的英文短语。该课内容涉及 LLM 应用安全、prompt injection、输出过滤和生产 guardrails，保留必要术语可以接受；但当前正文、表格、Mermaid 标签和测验解释中，许多说明性英文已经影响中文阅读连贯性。

## Findings

### P1 - Guardrails 课程中文文档仍是半翻译状态

文件：
- `phases/11-llm-engineering/12-guardrails/docs/zh-CN.md`
- `phases/11-llm-engineering/12-guardrails/quiz.zh-CN.json`

证据：
- 标题和开头混合 `LLM application`、`prompt injection`、`chatbot`、`agent`、`RAG pipeline`、`guardrails` 等英文短语，且不是只作为术语首次出现。
- 学习目标保留 `input guardrails`、`output guardrails`、`responses`、`policy violations`、`layered defense system`、`red-team prompt set`、`false positive/negative rate`。
- Mermaid 图示可见标签仍为英文，例如 `User Input`、`Input Validation`、`Pass`、`Block`、`Safe Response`、`Length Check`、`Rate Limit`。
- 表格列名和正文小节仍有大量英文：`Attack Type`、`Injection Point`、`Primary Defense`、`Tools of the Trade`、`Defense-in-Depth`、`Real Attack Case Studies`。
- quiz 文案和选项残留 `weights`、`system prompt`、`instructions`、`tokens`、`input guardrails`、`output validation`、`policy`、`red-team prompt set`。

影响：
- 这门课是 Phase 11 中安全防护链路的核心课。半中半英会让中文读者难以把攻击类型、防御层次和工具选择串起来，也会让本地测验看起来未完成。

### P2 - MCP 生产认证课仍是后续候选

文件：
- `phases/13-tools-and-protocols/18-mcp-auth-production/docs/zh-CN.md`

证据：
- 残留扫描仍显示 `Enrollment`、`JWKS Refresh`、`Audience-Pinned Tokens`、`dynamic client registration` 等混合内容。
- 该课规范引用密集，涉及 OAuth/OIDC、MCP spec、RFC 名称和 token/audience/JWKS 等术语，需要单独审查，避免误翻标准词。

影响：
- 不适合与本轮 guardrails 课混在一起处理；应作为下一轮优先候选。

## 第 4 轮建议修复范围

1. 只处理 `11-llm-engineering/12-guardrails`，保持每课独立提交。
2. 润色 `docs/zh-CN.md` 的标题、开头、学习目标、正文解释、表格列名、Mermaid 可见标签、步骤标题、交付成果、练习和术语表。
3. 润色 `quiz.zh-CN.json` 的问题、选项和解释，保持题目数量、`stage`、`correct` 索引和 JSON 结构不变。
4. 保留必要英文术语、API 名、模型名、库名、代码标识符、regex、prompt 示例和命令字符串。
5. 修改后运行单课 i18n 校验、全量 audit/i18n/site 测试，并用本地网页检查页面渲染、代码高亮和 quiz 区域。
