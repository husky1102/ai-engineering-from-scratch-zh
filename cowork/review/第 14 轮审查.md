# 第 14 轮审查

日期：2026-06-09

范围：
- 接续第 13 轮后，检查 Phase 14 相邻 Agent Workbench 课程。
- 重点检查 `39-reviewer-agent` 的中文翻译、quiz 结构、代码测试合同和本地网页可见状态。

## 结论

第 14 轮仍有问题，不能结束循环。

`phases/14-agent-engineering/39-reviewer-agent` 与上一课问题同型：中文文档保留大量普通英文，quiz 有 7 题且 pre 阶段过多，代码目录缺少 `tests/`。本课是 reviewer/gate/handoff 三课链条中的中间课，半翻译会让 builder、reviewer、rubric、bias mitigation 和 calibration set 的职责边界不清晰。

本地网页复验还发现运行代码面板直接显示 runtime manifest 的英文诊断 note，且加载态样式没有在内容填充后移除，导致运行说明继承全大写和宽字距。

## Findings

### P1 - Reviewer Agent 课程中文页面半翻译

文件：
- `phases/14-agent-engineering/39-reviewer-agent/docs/zh-CN.md`

证据：
- 标题仍为 `Reviewer Agent：将 Builder 与 Marker 分开`。
- Mermaid 可见标签仍为 `Builder Agent`、`Reviewer Agent`、`Human Sign-Off`。
- rubric 表格保留 `Problem fit`、`Scope discipline`、`Assumptions`、`Verification quality`、`Handoff readiness`，正文保留 `soft fail`、`hard fail`、`write access`、`clean change`、`right tests, wrong problem`。
- 生产模式保留 `Specialist pool`、`Bias mitigation`、`Calibration set`、`vibes`、`Hybrid norm`、`position bias`、`verbosity bias`、`self-preference`、`authority`、`consistent wins` 等普通解释性英文。
- 实际使用、交付成果、练习、术语表仍保留 `subagents`、`handoffs`、`Two-model pairing`、`human review`、`task close-outs`、`request more evidence`、`Reviewer rubric`、`Confidence floor` 等可见英文。

影响：
- 本课要教“builder 不应给自己打分”。半翻译会让 reviewer、rubric、gate 和 handoff 的边界变得含混，学生难以区分确定性 gate 与语义 reviewer 的职责。

### P2 - Reviewer Agent quiz 有 7 题且 pre 阶段过多

文件：
- `phases/14-agent-engineering/39-reviewer-agent/quiz.json`
- `phases/14-agent-engineering/39-reviewer-agent/quiz.zh-CN.json`

证据：
- 中英 quiz 都有 7 道题。
- 阶段分布为 `pre=2/check=3/post=2`。
- 课程合同要求 6 道题：1 pre、3 check、2 post。

影响：
- 测验结构不符合站点契约，也让课前检查负担偏重。

### P2 - Reviewer Agent 代码缺少 unittest

文件：
- `phases/14-agent-engineering/39-reviewer-agent/code/`

证据：
- `python3 -m unittest discover tests -v` 在本课 `code/` 下失败：`Start directory is not importable: 'tests'`。
- `code/main.py` 可运行，但没有 5+ stdlib unit tests 覆盖 reviewer scoring、verdict thresholds 和 report writing。

影响：
- 违反每课至少 5 个 unit tests 的课程合同。
- reviewer rubric 的阈值和边界没有可重复验证，后续修改容易悄悄改变 hard/soft/pass 判定。

### P2 - 中文网页运行面板显示英文 runtime note

文件：
- `site/lesson.html`
- `site/i18n.js`
- `site/i18n/en.json`
- `site/i18n/zh-CN.json`

证据：
- 浏览器打开 `lesson.html?path=phases/14-agent-engineering/39-reviewer-agent&lang=zh-CN` 后，运行代码面板显示 `PYTHON USES PACKAGES OUTSIDE THE BROWSER ALLOWLIST`。
- `runtime-manifest.json` 中的诊断 note 是英文原始字符串，`renderCodePanel` 直接 `escapeHtml(note)` 输出，没有经过 `t(...)`。
- 进一步检查发现 `#codeContent` 内容加载后仍保留 `panel-loading` class，导致正文说明、路径和入口继承 loading 态的 `text-transform: uppercase` 与宽字距。

影响：
- 中文网页的运行能力说明不自然，且全大写会让路径、入口和说明看起来像异常状态。
- 同类 runtime note 还会影响其他课程的中文页面。

## 第 14 轮建议修复范围

1. 润色 `docs/zh-CN.md` 的标题、导语、学习目标、Mermaid 标签、rubric 表格、核心概念、生产模式、实际使用、练习、术语表和延伸阅读说明。
2. 保留 reviewer agent、builder、rubric、verification gate、handoff、LLM judge、calibration set、confidence 等必要术语；将普通解释性英文改为自然中文或中文优先的双语术语。
3. 将中英 quiz 收敛到 6 题，并调整为 1 pre、3 check、2 post；中文与英文源保持相同 `stage`、选项数量和 `correct` 索引。
4. 补齐本课 stdlib unittest，覆盖每个 scoring 函数、verdict thresholds、report writing demo 的关键结构。
5. 运行单课 i18n 校验、Python demo、unittest、全量验证和本地网页复验；本课修复作为单独 lesson commit，并清理 `review_report_*.json`。
6. 将 runtime note 映射到 i18n key，补齐中英语言包；运行面板和交付物面板加载完成后移除 `panel-loading` class，并 bump `i18n.js` cache-buster。
