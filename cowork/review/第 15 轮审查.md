# 第 15 轮审查

日期：2026-06-09

范围：
- 接续第 14 轮后，检查 Phase 14 Agent Workbench 收尾课程。
- 重点检查 `40-multi-session-handoff` 的中文翻译、quiz 结构、代码测试合同和本地网页风险。

## 结论

第 15 轮仍有问题，不能结束循环。

`phases/14-agent-engineering/40-multi-session-handoff` 与前两课同属 reviewer/gate/handoff 链条，存在同型合同缺口：中英 quiz 都是 7 题且 pre 阶段过多，代码目录没有 `tests/`。中文文档整体可读，但还保留了一批面向读者的普通英文标签，尤其是 Mermaid、清理检查表、生产模式小标题和术语表，影响中文页面的自然度。

## Findings

### P1 - Multi-Session Handoff 中文页面仍有普通英文标签

文件：
- `phases/14-agent-engineering/40-multi-session-handoff/docs/zh-CN.md`

证据：
- Mermaid 可见标签仍有 `Next Session`。
- 清理检查表保留 `Working tree`、`Temp artifacts`、`Tests`、`Feature board`、`Branch`。
- 生产模式标题保留 `Session-end hook`、`PR template`、`Cross-agent handoff`，正文保留 `lingua franca`。
- 关键术语表保留 `Handoff packet`、`Next action`、`Feedback trim`、`Status report`、`Verdict pointer` 作为第一列可见术语。
- 练习中保留 `questions for the human`、`next session prereqs`，延伸阅读说明中保留 `distributed-systems framing`、`handoff-oriented prompts`。

影响：
- 本课要教“交接包是生成出来的恢复工件”。这些标签是读者会直接看到的结构性文字，不是代码字段；保留英文会让中文课程读起来像半翻译草稿。

### P2 - Multi-Session Handoff quiz 有 7 题且 pre 阶段过多

文件：
- `phases/14-agent-engineering/40-multi-session-handoff/quiz.json`
- `phases/14-agent-engineering/40-multi-session-handoff/quiz.zh-CN.json`

证据：
- 中英 quiz 都有 7 道题。
- 阶段分布为 `pre=2/check=3/post=2`。
- 课程合同要求 6 道题：1 pre、3 check、2 post。

影响：
- 测验结构不符合站点契约，也延续了 Phase 14 末尾几课的同型 pre 重复问题。

### P2 - Multi-Session Handoff 代码缺少 unittest

文件：
- `phases/14-agent-engineering/40-multi-session-handoff/code/`

证据：
- `python3 -m unittest discover tests -v` 在本课 `code/` 下失败：`Start directory is not importable: 'tests'`。
- `code/main.py` 可运行并会写出 `handoff.md`、`handoff.json`，但没有 5+ stdlib unit tests 覆盖 feedback trimming、risk derivation、payload shape、fallback next action 和 demo output。

影响：
- 违反每课至少 5 个 unit tests 的课程合同。
- handoff packet 的核心字段和裁剪规则没有可重复验证，后续修改容易破坏恢复工件。

## 第 15 轮建议修复范围

1. 润色 `docs/zh-CN.md` 的 Mermaid 标签、清理检查表、生产模式、实际使用、练习、术语表和延伸阅读说明。
2. 保留 `summary`、`next_action`、`handoff.md`、`handoff.json`、`feedback_record.jsonl`、`active | superseded | archived` 等代码字段或 schema 值；将普通解释性英文改为中文或中文优先的双语术语。
3. 将中英 quiz 收敛到 6 题，并调整为 1 pre、3 check、2 post；中文与英文源保持相同 `stage`、选项数量和 `correct` 索引。
4. 补齐 stdlib unittest，覆盖 `trim_feedback`、`derive_risks`、`generate_handoff`、fallback `next_action` 和 `main()` 写出的 JSON/Markdown。
5. 运行单课 i18n 校验、Python demo、unittest、全量验证和本地网页复验；本课修复作为单独 lesson commit，并清理 `handoff.md`、`handoff.json`。
