## Findings

### P0 - `site/` 作为静态根时课程页无法加载正文

证据：
- 启动：`cd site && python3 -m http.server 4173 --bind 127.0.0.1`
- 打开：`http://127.0.0.1:4173/lesson.html?path=phases/01-math-foundations/01-linear-algebra-intuition`
- 页面显示：`未找到课程`，提示无法加载 `phases/01-math-foundations/01-linear-algebra-intuition`。

原因：
- [site/lesson.html](/Users/lolita/mine/work/ai-engineering-from-scratch-zh/site/lesson.html:2072) 的正文候选路径是 `../${path}/docs/...`。
- Vercel 配置的 `outputDirectory` 是 `site`，典型本地入口也是 `site` 目录作为静态根；此时 `../phases/...` 跳出静态根，浏览器拿不到课程源文件。

影响：
- 中文译文存在，但站点根入口无法展示。
- `quiz.zh-CN.json` 无法加载。
- Pyodide 运行按钮无机会初始化。
- 输出/代码卡片的 `../phases/...` 链接在同类部署入口下也会失效。

### P1 - 中文 lesson 页面底部重复出现英文通用测验

证据：
- 仓库根入口能加载中文正文后，页面同时存在中文课前/课后测验，以及底部英文 `QUESTION 1 OF 3` 通用测验。
- [site/lesson.html](/Users/lolita/mine/work/ai-engineering-from-scratch-zh/site/lesson.html:3302) 总是调用 `renderQuizPanel(container)`。
- [site/lesson.html](/Users/lolita/mine/work/ai-engineering-from-scratch-zh/site/lesson.html:3538) 之后的 `getQuizQuestions()` 是按路径关键词生成的英文通用题，不来自 lesson 的 `quiz.zh-CN.json`。

影响：
- 用户会看到两套测验，且一套是英文、泛化、与本课中文题库不一致。
- 测验结果和学习完成状态容易混淆。

### P1 - AI 面板仍有硬编码英文文案

证据：
- 同一中文页面的 AI 面板标题包括：
  - `TEST YOUR UNDERSTANDING`
  - `LEARNING PATH`
  - `CONTINUE LEARNING`
- 硬编码位置包括 [site/lesson.html](/Users/lolita/mine/work/ai-engineering-from-scratch-zh/site/lesson.html:3709) 和 [site/lesson.html](/Users/lolita/mine/work/ai-engineering-from-scratch-zh/site/lesson.html:3782)。

影响：
- 中文页面体验割裂。
- 翻译本身虽覆盖，但站点 chrome 没有完全本地化。

### P2 - `text` 公式块被通用代码高亮误染色

证据：
- 中文课程中有 621 个 `text` fenced block 含公式/伪公式，例如：
  - `cosine_sim(a, b) = dot(a, b) / (||a|| * ||b||)`
  - `MSE = (1/n) * sum((y_pred - y_true)^2)`
- 页面实测中 `text` block 也会进入 `highlightSyntax()` 的默认关键字分支，出现 `hasSyn: true`。
- [site/lesson.html](/Users/lolita/mine/work/ai-engineering-from-scratch-zh/site/lesson.html:2793) 对未知语言使用通用关键字规则。

影响：
- 公式块视觉上像代码被随机着色，不像公式/说明。
- 这不影响可读文本内容，但不满足“公式渲染正常”的体验要求。

### P2 - Notebook 链接在 `site/` 静态根下多了一层 `site/`

证据：
- 修复资源链接后复测代码面板，`打开 NOTEBOOK` 的 href 为 `site/notebooks/phases/.../lesson.ipynb`。
- 在 `site/` 作为静态根时，正确 URL 应为 `notebooks/phases/.../lesson.ipynb`。
- 来源是 [scripts/build_notebooks.py](/Users/lolita/mine/work/ai-engineering-from-scratch-zh/scripts/build_notebooks.py:189) 写入 runtime manifest 时保留了仓库相对路径。

影响：
- Lab 链接可用，但直接打开 notebook 文件会 404。

### P2 - 课程契约与历史翻译策略存在 quiz 张力

证据：
- `phases/**/docs/en.md`：503 个。
- `phases/**/quiz.json`：338 个。
- `docs_without_quiz`：165 个。
- 历史翻译会话要求：“如果某个 lesson 没有 `quiz.json`，就只处理 `docs/zh-CN.md`，不要为了补齐而新造 quiz。”
- 当前 AGENTS lesson contract 写的是每课应有 `quiz.json`。

影响：
- 这不是中文翻译遗漏，因为中文 quiz 与源 quiz 一一对应。
- 但如果删除英文通用测验面板，165 个无源 quiz 的课程仍没有课程内测验。
- 该问题需要后续轮次单独决策：补源 `quiz.json`，还是更新契约/审计规则。

## 第 1 轮建议修复范围

本轮先修网页关键路径，不批量改 165 课题库：

1. 在 `site/build.js` 中生成 `site/content/phases/...`，复制课程页面需要的 `docs/`、`quiz*.json`、`code/`、`outputs/` 等静态源文件。
2. 在 `.gitignore` 中忽略 `site/content/`，因为它由 `site/build.js` 在本地/CI/Vercel 构建时生成。
3. 在 `site/lesson.html` 中优先从 `content/${path}/...` 加载正文和 quiz，保留 `../${path}/...` 作为仓库根本地入口的回退。
4. 在 `scripts/build_lesson_assets.py` 中把 code/output URL 改成 `content/...`，使部署静态根中的“查看源码/打开文件”链接可用。
5. 移除底部英文通用 quiz panel，保留 lesson 自带 `quiz.json` / `quiz.zh-CN.json` 注入的分阶段测验。
6. 将学习路径/继续学习面板的硬编码英文切到 i18n key。
7. `text` / `console` 等纯文本块不做语法高亮，避免公式块被代码关键字染色。
8. 修正 notebook manifest URL，把 `site/notebooks/...` 规范为站点内 `notebooks/...`。
