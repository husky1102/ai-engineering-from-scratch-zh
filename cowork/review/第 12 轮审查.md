# 第 12 轮审查

日期：2026-06-09

范围：
- 复查第 11 轮后的翻译残留。
- 检查 Phase 14 后段 Agent Workbench 课程的中文页面和 quiz 结构。
- 继续检查本地网页渲染、代码高亮、运行代码面板和 quiz 结构。

## 结论

第 12 轮仍有问题，不能结束循环。

`phases/14-agent-engineering/37-runtime-feedback-loops` 是当前最明确的问题课程。中文页面标题、Mermaid、表格、概念段、生产模式、术语表和 quiz 都保留大量普通英文；quiz 还有 7 题，超过课程合同要求的 6 题。

## Findings

### P1 - Runtime Feedback Loops 课程中文页面半翻译

文件：
- `phases/14-agent-engineering/37-runtime-feedback-loops/docs/zh-CN.md`

证据：
- 标题仍为 `Runtime Feedback Loops`。
- Mermaid 图中可见标签仍为 `Agent Loop`、`stdout / stderr / exit / duration`、`Verification Gate`。
- 表格和正文保留 `shell expansion surprise`、`slow probes`、`runaway processes`、`replay`、`timestamp`、`agent note`、`feedback records`、`inline`、`re-runs tail`。
- 生产模式保留 `redact`、`redaction pass`、`foot-gun`、`attacker`、`runtime cost`、`loader call`、`retry chains`、`parent-command id`、`successes`、`audit`。
- 术语表保留 `Feedback record`、`Run log`、`Tail truncation`、`Trim the log`、`Refuse-on-null`、`Agent note`、`Telemetry split` 等普通英文。

影响：
- 本课讨论的是“代理不能凭想象声称成功”的核心工作台机制；半翻译会削弱读者对 feedback、telemetry、verification gate、retry chain 这些概念边界的理解。

### P2 - Runtime Feedback Loops quiz 有 7 题且 pre 阶段过多

文件：
- `phases/14-agent-engineering/37-runtime-feedback-loops/quiz.json`
- `phases/14-agent-engineering/37-runtime-feedback-loops/quiz.zh-CN.json`

证据：
- 中英 quiz 都有 7 道题。
- 阶段分布为 `pre=2/check=3/post=2`。
- 课程合同要求 6 道题：1 pre、3 check、2 post。

影响：
- 页面可显示测验，但数量不符合课程合同；多出来的 pre 题打乱了课前检查的节奏。

### P3 - 本地运行面板把 stdlib import 显示成需要安装的包

文件：
- `site/lesson.html`

证据：
- 本地网页运行面板显示 `包: __FUTURE__, SHLEX, SUBPROCESS, UUID`。
- 这些都是 Python 标准库 import，不应该显示为运行本课需要安装的 package。

影响：
- 读者会误以为需要安装标准库模块，降低本地运行面板可信度。

## 第 12 轮建议修复范围

1. 润色 `docs/zh-CN.md` 的标题、导语、学习目标、Mermaid 标签、表格、核心概念、生产模式、实际使用、练习、术语表和延伸阅读说明。
2. 保留 feedback、telemetry、runner、record、JSONL、stdout/stderr、exit_code、duration、verification gate、redaction、rotation、retry chain、parent_command_id、Claude Code、LangGraph、CI 等必要术语；将普通解释性英文改为自然中文。
3. 将中英 quiz 收敛到 6 题，并调整为 1 pre、3 check、2 post；中文与英文源保持相同 `stage`、选项数量和 `correct` 索引。
4. 修复本地运行面板，不再显示 Python 标准库 import 为 package。
5. 运行单课 i18n 校验、Python demo、全量验证和本地网页复验；本课修复作为单独 lesson commit，并清理 demo 生成的 `feedback_record.jsonl*`。
