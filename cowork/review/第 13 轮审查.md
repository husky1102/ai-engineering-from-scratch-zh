# 第 13 轮审查

日期：2026-06-09

范围：
- 复查第 12 轮后的翻译残留。
- 检查 Phase 14 相邻 Agent Workbench 课程的中文页面和 quiz 结构。
- 继续检查本地网页渲染、代码高亮、运行代码面板和 quiz 结构。

## 结论

第 13 轮仍有问题，不能结束循环。

`phases/14-agent-engineering/38-verification-gates` 是当前最明确的问题课程。中文页面标题、Mermaid、表格、生产模式、术语表和 quiz 都有明显普通英文残留；quiz 也有 7 题，超过课程合同要求的 6 题。执行验证时还发现本课没有 `code/tests/`，且 coverage regression 的 1 个百分点边界被浮点误差误判为 block。浏览器复验时，站点运行面板还把 Python stdlib 的 `hmac` 显示成外部包。

## Findings

### P1 - Verification Gates 课程中文页面半翻译

文件：
- `phases/14-agent-engineering/38-verification-gates/docs/zh-CN.md`

证据：
- 标题仍为 `Verification Gates`。
- Mermaid 图中可见标签仍为 `Diff`、`Reviewer Agent`、`refuse done + surface to human`。
- 表格与正文保留 `acceptance commands`、`forbidden writes`、`off-scope writes`、`Touched files`、`verdict`、`LLM judges`、`status`、`task close-out`、`fork 真相来源`。
- 生产模式保留 `Defense-in-depth`、`Pre-commit hook`、`CI status check`、`pre-tool authz hook`、`model-side skill`、`model-judge`、`nuance`、`Signed override log`、`override theater`、`Coverage floor`、`strict-on-everything`。
- 术语表保留 `Verification gate`、`Hard fail`、`signed entries`、`Acceptance command`、`Source of truth` 等普通英文。

影响：
- 本课定义“完成与否”的确定性边界。半翻译会让 gate、reviewer、override、coverage floor 和 strict mode 的职责边界变模糊。

### P2 - Verification Gates quiz 有 7 题且 pre 阶段过多

文件：
- `phases/14-agent-engineering/38-verification-gates/quiz.json`
- `phases/14-agent-engineering/38-verification-gates/quiz.zh-CN.json`

证据：
- 中英 quiz 都有 7 道题。
- 阶段分布为 `pre=2/check=3/post=2`。
- 课程合同要求 6 道题：1 pre、3 check、2 post。

影响：
- 页面可显示测验，但数量不符合课程合同；多出来的 pre 题打乱了课前检查的节奏。

### P2 - Verification Gates 代码缺少 unittest，coverage 边界误判

文件：
- `phases/14-agent-engineering/38-verification-gates/code/main.py`
- `phases/14-agent-engineering/38-verification-gates/code/tests/`

证据：
- `python3 -m unittest discover tests -v` 在本课 `code/` 下失败：`Start directory is not importable: 'tests'`。
- `python3 phases/14-agent-engineering/38-verification-gates/code/main.py` 输出第一例 `T-001: passed=False`，原因是 `0.85 - 0.84` 的浮点误差让“刚好 1 个百分点下降”触发 `coverage.regression` block。
- 文档说 demo 包含 `clean pass`，但实际第一例不是 clean pass。

影响：
- 违反每课至少 5 个 stdlib unit tests 的课程合同。
- 学生运行 demo 时会看到与课程叙述相反的结果，削弱 verification gate 对边界条件的教学可信度。

### P2 - 本地网页运行面板误显 stdlib 包

文件：
- `scripts/detect_runtimes.py`
- `site/lesson.html`

证据：
- 本地页面 `lesson.html?path=phases/14-agent-engineering/38-verification-gates&lang=zh-CN` 的运行面板显示 `包: HMAC`。
- `hmac` 是 Python 标准库，不应被展示为外部 runtime package。
- detector 还会递归扫描 `code/tests/`，新增测试后会把测试里的 `main`、`tempfile` 污染进 runtime packages。

影响：
- 运行面板让 stdlib-first 课程看起来有外部依赖。
- 测试文件会污染 runtime manifest，导致网页运行信息不稳定。

## 第 13 轮建议修复范围

1. 润色 `docs/zh-CN.md` 的标题、导语、学习目标、Mermaid 标签、表格、核心概念、生产模式、实际使用、练习、术语表和延伸阅读说明。
2. 保留 verification gate、scope contract、feedback log、rule report、diff、verdict、reviewer、CI、override、coverage_floor、strict mode 等必要术语；将普通解释性英文改为自然中文。
3. 将中英 quiz 收敛到 6 题，并调整为 1 pre、3 check、2 post；中文与英文源保持相同 `stage`、选项数量和 `correct` 索引。
4. 补齐本课 stdlib unittest，覆盖 acceptance、scope、coverage、strict 和 signed override；修复 coverage regression 浮点边界。
5. 修复本地网页运行面板的 stdlib 包过滤，并让 runtime detector 跳过 `code/tests/`。
6. 运行单课 i18n 校验、Python demo、全量验证和本地网页复验；本课修复作为单独 lesson commit，并清理 demo 生成的 `verification_report_*.json` / `overrides.jsonl`。
