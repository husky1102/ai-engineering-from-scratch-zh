# 第 18 轮审查

日期：2026-06-09

范围：
- 接续第 17 轮后，检查自动扫描出的下一课：`00-setup-and-tooling/03-gpu-setup-and-cloud`。
- 重点检查中文翻译自然度、quiz 结构、代码入口契约和本地网页可运行性。

## 结论

第 18 轮仍有问题，不能结束循环。

`phases/00-setup-and-tooling/03-gpu-setup-and-cloud` 的中英 quiz 仍是旧分布：5 题、`pre=2/post=3`、没有 `check` 阶段，也缺少 `lesson` 和 `title` 顶层字段。课程声明语言为 Python，且存在 `code/gpu_check.py`，但没有 `main.py`；本地网页运行面板按 `main.*` 识别入口，因此这课会被当作静态阅读，不能直接运行代码。中文文档整体可读，但概念框和 Colab 操作步骤仍保留普通英文界面/标签，作为中文课文不够自然。

## Findings

### P1 - GPU 课程 quiz 是旧分布

文件：
- `phases/00-setup-and-tooling/03-gpu-setup-and-cloud/quiz.json`
- `phases/00-setup-and-tooling/03-gpu-setup-and-cloud/quiz.zh-CN.json`

证据：
- 中英 quiz 都只有 5 道题。
- 阶段分布为 `pre=2/post=3`，没有 `check`。
- quiz 文件缺少 `lesson` 和 `title` 顶层字段。
- 课程合同要求 6 道题：1 pre、3 check、2 post。

影响：
- 站点可以渲染旧测验，但课程节奏不符合当前契约，缺少课中检查。

### P1 - Python 代码没有 canonical `main.py` 入口和测试

文件：
- `phases/00-setup-and-tooling/03-gpu-setup-and-cloud/code/gpu_check.py`

证据：
- `code/` 目录只有 `gpu_check.py`，没有 `main.py`。
- 站点运行面板和 lesson contract 都以 `main.*` 作为语言入口。
- 该课没有 `code/tests/`，无法锁定无 GPU、无 PyTorch、CUDA 可用等分支行为。
- 现有脚本默认会在检测到 CUDA 后运行 4000x4000 矩阵乘法；这适合本地实验，但不适合作为网页的一键运行默认路径。

影响：
- 本地网页无法直接运行本课代码。
- 在没有 PyTorch 或 GPU 的机器上，缺少可测试、可重复的教学诊断行为。

### P2 - 中文文档普通标签仍偏英文

文件：
- `phases/00-setup-and-tooling/03-gpu-setup-and-cloud/docs/zh-CN.md`

证据：
- 概念框保留 `Your options`、`Cost`、`Setup`、`Best for` 等普通英文标签。
- Colab 步骤保留 `Runtime > Change runtime type > T4 GPU`，没有中文说明。

影响：
- 这些不是命令或 API 名称，而是课文解释标签；中文页面应中文优先，并保留必要英文产品名或菜单名。

### P2 - 运行代码面板加载后副标题仍显示 loading

文件：
- `site/lesson.html`
- `site/tests/runtime_notes.test.mjs`

证据：
- Browser QA 中，`#codeContent` 已经加载出 `code/main.py`、`包: torch`、运行命令和源码卡片。
- 同一个运行代码面板的 `.ai-panel-subtitle` 仍显示 `正在加载运行能力...`。

影响：
- 用户会误以为运行能力仍在加载，和实际已加载的运行/源码卡片状态不一致。

## 第 18 轮建议修复范围

1. 新增 `code/main.py`，把 GPU 诊断拆成可测试函数；默认运行只做轻量检测，`--benchmark` 才运行矩阵乘法。
2. 新增 `code/tests/test_main.py`，覆盖无 PyTorch、CPU-only、CUDA 可用、fp16 参数估算和 CLI 默认退出。
3. 将 `gpu_check.py` 改成兼容 wrapper，转发到 `main.py`。
4. 润色中英文档，更新 canonical command 为 `python3 main.py`，中文概念框中文优先。
5. 将中英 quiz 改为 6 题，阶段分布为 `pre=1/check=3/post=2`，补齐顶层 `lesson` 和 `title`。
6. 更新 site runtime/source manifest，让本地网页指向 `code/main.py`，并确保 `code/tests/` 不出现在源码面板。
7. 修复运行代码面板副标题，资产加载完成后显示运行类型或源码状态，而不是继续显示 loading。
8. 运行单课代码测试、单课 i18n、全量校验和本地网页复验；本课修复作为单独 lesson commit。
